# SDD — Экономия token-pilot в долларах + таблица цен моделей Claude

- **Дата:** 2026-06-18
- **Класс:** feature
- **Область:** только loom-host (UI-слой `web/`)
- **Задача (journal):** tj-tphfy0jgtc

## 1. Зачем (проблема)

Сейчас экономия от token-pilot видна только в токенах: на борде (`Tokens.tsx`)
и в строке стоимости задачи (`CostBar.tsx`) пишется «saved ≈ N tokens, X%».
В деньгах экономии не видно, потому что нет таблицы цен по моделям. Бизнес-смысл
token-pilot — экономия денег, а не абстрактных токенов, поэтому рядом с «tokens
saved» нужно показать «$ saved».

## 2. Что делаем (scope)

1. Отдельный модуль с **публичными ценами моделей Claude** (input / output $ за
   1M токенов).
2. Пересчёт сэкономленных токенов в доллары: **`$ saved = saved_tokens × входная
   цена за токен`** (token-pilot экономит в основном входные токены).
3. Показать «$ saved» рядом с «tokens saved»:
   - **per-task** — в `CostBar.tsx` (строка «Cost» в карточке задачи);
   - **по борду** — в `Tokens.tsx` (итоги + таблицы by subscription / by session).

### Вне scope (не трогаем)

- **Реальная стоимость** задачи остаётся провайдерской: строка `aimux/spent`
  (`total_cost_usd` из `live-session.ts`). Таблица цен — только для пересчёта
  *экономии* в $ и как сверка (cross-check), не замена настоящему счёту.
- Существующий учёт токенов: `tokens-per-task.ts`, `token-pilot/adapter.ts`,
  `cost-recorder.ts`, таблица `cost_rollups`, эндпоинт `/api/tokens` —
  без изменений. Никаких новых строк в `cost_rollups`.
- Селектор моделей / настройка цены пользователем — не просят.

## 3. Ключевые решения и допущения

| # | Решение | Обоснование |
|---|---------|-------------|
| D1 | Модуль цен и пересчёт живут в **`web/src/`** (клиент). | `web/` не импортирует из `src/core/` (проверено: `git grep` — 0 совпадений). Обе точки показа $ — в `web/`. Единый источник, ноль изменений в серверном учёте. |
| D2 | `$ saved` считается **на клиенте** в обеих точках. | Борд получает токены из `/api/tokens`, per-task — из `cost_rollups`. Оба уже на клиенте → серверные эндпоинты не трогаем. |
| D3 | Допущение: у сэкономленных токенов **нет привязки к модели** (token-pilot её не знает). Берём входную цену одной **дефолтной модели**. | Данные `saved` модель-агностичны. Цифра приблизительная → помечаем «≈». Дефолт — константа в модуле (текущая Opus). |
| D4 | Переиспользуем `formatUsd()` из `web/src/ui.ts`. | Уже есть форматтер $; не плодим дубликаты. |

## 4. Дизайн

### 4.1 Новый модуль `web/src/pricing.ts`

Публичная таблица цен Claude (USD за 1M токенов, input / output) + хелперы:

```ts
export interface ModelPrice { id: string; label: string; inputPerMTok: number; outputPerMTok: number; }

// Публичные прайсы Claude (USD / 1M токенов). Источник — официальный прайс-лист.
export const MODEL_PRICES: ModelPrice[] = [ /* opus / sonnet / haiku ... */ ];

// Модель по умолчанию для пересчёта экономии (token-pilot не знает реальную модель).
export const DEFAULT_PRICING_MODEL = "<opus-id>";

// saved_tokens → $ по входной цене модели. Неизвестная модель → дефолт.
export function savedTokensToUsd(savedTokens: number, modelId?: string): number;
```

- Таблица — единственный источник правды по ценам. Output-цена включается в
  таблицу (задача просит «input/output»), но в пересчёте экономии используется
  только `inputPerMTok` (D3).
- Чистые функции, без побочных эффектов и сети.

### 4.2 `web/src/ui.ts` — `summarizeCosts()` / `CostSummary`

- Добавить в `CostSummary.tokens` поле `savedUsd: string | null` (строка вида
  `"$0.42"` через `formatUsd`, либо `null` если `saved <= 0`).
- Считать: `savedUsd = formatUsd(savedTokensToUsd(savedValue))`.
- Существующие поля (`used`, `saved`, `savedPct`, `spend`, …) — без изменений.

### 4.3 `web/src/components/CostBar.tsx`

- В блоке `cost-saved` дописать `$ saved` рядом с токенами:
  `· {s.tokens.saved} saved ({s.tokens.savedPct}%) · {s.tokens.savedUsd} saved`
  (точная формулировка/порядок — по месту, текст английский, «≈» сохраняем).

### 4.4 `web/src/components/Tokens.tsx`

- В итоговом блоке «Saved by token-pilot» добавить сумму `$ saved`.
- В таблицах «By subscription» и «By session» добавить колонку `$ saved`
  (через `savedTokensToUsd(row.saved)` + `formatUsd`).
- Заголовки/подписи — английские.

## 6. Тесты

- Существующие тесты `tokens-per-task` и компонентов — должны остаться зелёными
  без правок (мы их не трогаем).

## 7. Критерии приёмки

1. На борде (`Tokens.tsx`) рядом с сэкономленными токенами видна сумма в `$`
   (итог + обе таблицы).
2. В карточке задачи (`CostBar.tsx`) рядом с «N saved (X%)» видно «$ saved».
3. Реальная стоимость задачи (`aimux/spent`) и числа `used/saved` не изменились;
   `cost_rollups` и `/api/tokens` не тронуты.
4. Цены лежат в отдельном модуле; пересчёт использует входную цену.
5. `bun run check:ds` (web и host) — зелёный.
6. `web` tsc (`tsc --noEmit`) и host tsc (`build:host`) — зелёные.
7. Новые юнит-тесты проходят; полный прогон тестов зелёный.

## 8. Блок-радиус / риски

- Малый: изменения локализованы в `web/src/` (1 новый модуль + правки `ui.ts`,
  `CostBar.tsx`, `Tokens.tsx` + 2 файла тестов). Серверный код и БД не затронуты.
- Риск устаревания цен — это таблица публичных цен; держим в одном модуле с
  пометкой источника, обновление = правка одной таблицы.
- Приблизительность ($ по дефолтной модели) — помечаем «≈», как уже принято в UI.

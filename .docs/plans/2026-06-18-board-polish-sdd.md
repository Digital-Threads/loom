## 1. Цель

Доска задач работает, но крайние состояния выглядят сыро. Привести их к
дизайн-системе (DS), не трогая поведение:

1. Показать нормальный **empty-state всей доски**, когда задач нет совсем или
   ни одна не проходит под выбранный фильтр проекта (сейчас рисуется ряд пустых
   колонок с прочерком «—»).
2. Заменить **пустую колонку** (`<div className="skip">—</div>`) на аккуратную
   подсказку-дропзону по DS («Drop a task here»).
3. Проверить **типографику счётчиков** колонок (`span.n`).
4. Выровнять **плотность карточек** по 4px-сетке/токенам.

Сохранить без изменений: drag-and-drop, подсветку колонки `col-over`, кнопки
удаления/остановки, обработку клавиатуры и `aria`-атрибуты.

Границы: только `loom-host/web`. UI на английском. `npm run check:ds` зелёный,
`tsc -b` ок, существующие тесты не падают, добавлен тест на Board.

## 3. Проектные решения

### 3.1 Empty-state доски через расширение StateView (выбрано)

Добавить в `StateView` **необязательный** слот действия:

```tsx
export function StateView({ kind, msg, action }: {
  kind: "loading" | "empty" | "error";
  msg?: string;
  action?: React.ReactNode;
}) { ... }
```

`action` рендерится под текстом (по центру) только для непустого значения.
Опциональность → все 17 текущих вызовов остаются рабочими без правок.

Отвергнуто: верстать CTA-блок локально в Board — дублирует layout пустого
состояния и расходится с общим компонентом; задача прямо просит «StateView
kind=empty с текстом и CTA».

### 3.2 Board получает onNew

Board принимает новый **необязательный** проп `onNew?: () => void`. В `App.tsx`
прокидывается тот же обработчик, что у кнопки «+ New» (`() => setShowNew(true)`).
CTA в empty-state рендерится только если `onNew` передан.

### 3.3 Условие и тексты пустого состояния

- Доска считается пустой, когда `cols` загружены и сумма видимых карточек по
  всем колонкам = 0: `cols.every(c => c.cards.filter(visible).length === 0)`.
- Тексты различать:
  - нет задач совсем (фильтр пуст или применён, но и без фильтра пусто): напр.
    «No tasks yet» + CTA «+ New task».
  - есть задачи, но ни одна не проходит под фильтр проекта: напр. «No tasks in
    this project» (CTA создания тоже уместен).
  - Различаем по тому, есть ли вообще карточки в `cols` без учёта фильтра:
    `cols.some(c => c.cards.length > 0)`.

### 3.4 Пустая колонка → подсказка дропзоны

Заменить `<div className="skip">—</div>` на
`<div className="dropzone">Drop a task here</div>`. Новый стиль `.dropzone` по
DS: приглушённый текст (`var(--dim)`/`var(--mut)`), пунктирная рамка
(`1px dashed var(--line)`), отступы и радиус из токенов (`--space-*`, `--r-md`),
выравнивание по центру. Только токены/разрешённые литералы — без hex.

### 3.5 Типографика счётчика

Проверить `.col h2 .n`: оставить `font-size:11px`, перевести скругление на токен
`--r-pill` вместо «магических» 20px (визуально эквивалентно — pill). Отступы при
необходимости на `--space-*`. Изменение косметическое и безопасное.

### 3.6 Плотность карточек по 4px-сетке

Самая визуально чувствительная правка — делать минимально, только на токены,
ближайшие к текущим значениям, чтобы вид почти не «прыгнул»:

| Сейчас | Токен (4px) |
|---|---|
| `.card` padding `11px 12px 11px 13px` | `var(--space-6)` (12px) единообразно, левый край компенсирует акцентная полоса `::before` 2px |
| `.col .stack` `padding:9px;gap:9px` | `var(--space-4)`…`--space-5` (8–10px) |
| `.card .t` `margin-bottom:8px` | `var(--space-4)` (8px) |
| `.card .meta` `gap:5px` | `var(--space-2)`/`--space-3` (4–6px) |

Точные значения подбираются на impl с визуальной сверкой; цель — кратность 4px и
использование токенов, без заметного сдвига макета.

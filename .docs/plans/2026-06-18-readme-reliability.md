## 1. Цель

Добавить в корневой `README.md` короткий раздел **Reliability** (3–5 строк, по-английски),
который документирует два уже реализованных поведения. Новый код поведения не пишется —
только описание существующего.

## 2. Контекст и обоснование

Оба поведения работают в продукте, но в README про них ничего нет. Пользователю важно знать,
что Loom переживает исчерпание лимита подписки и что Stop действительно останавливает агента.
Это также «живая проверка» инфраструктуры пайплайна на маленькой реальной задаче.

Подтверждение, что поведения реальны (источник формулировок, эти файлы НЕ правим):

- `src/core/automation/auto-fallback.ts` — `pickFallbackProfile()` выбирает другую подписку
  «с запасом»; `shouldAutoFallback()` срабатывает после grace-окна (по умолчанию 60 c).
  Если свободных подписок нет — задача честно паркуется, а не делает вид, что переключилась.
- `src/core/automation/session-relocate.ts` — перенос задачи на другой аккаунт с продолжением
  **той же** сессии.
- Коммит `952d324` «fix: reliable Stop + honest rate-limit status» — источник обоих поведений.

## 3. Объём работ

### В объёме
- Одна правка: `README.md`. Новый раздел `## Reliability` после `## How it works`,
  перед `## Stack`.
- Содержание (смысл, не дословно):
  1. **Rate-limit recovery** — при исчерпании лимита подписки текущего аккаунта Loom
     переключается на другой аккаунт с запасом и продолжает ту же сессию; есть авто-fallback,
     срабатывающий после короткой паузы, если пользователь не выбрал замену сам.
  2. **Stop** — надёжно останавливает агента и не возобновляет его автоматически.

### Вне объёма
- Любые изменения кода поведения, конфигов, тестов.
- Любые правки README за пределами нового раздела.
